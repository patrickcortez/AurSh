# Contribution

Thank for your interest in contributing to **AurSh**

To keep the project maintanable and scalable:

When contributing to **AurSh**. You must follow these simple guidelines and rules:

- When writing a new feature, Fixing buggy/inefficient parts of the codebase:
    Make sure to create a branch named after the issue. No changes must be made to *master*
    without permission.

- Be respectful and friendly, We do not tolerate harsh and immoral behavior. Offenders will be
    dealth with swiftly.

- Read the [docs](/docs) and familiarize yourself before writing any changes to the code base.

- When adding a new feature/functionality make sure to document its entire purpose and architecture.

- Make sure your pull requests less verbose and straight to the point.

- The use of AI is acceptable but make sure you can explain the code you wrote.

Following these rules and guidelines is paramount in contributing to the project.
If you want to be a contributor feel free to contact me.

## Notes

Before opening a pull request, make sure:

- Build Successfully
- Does not break existing funcionality
- Code is cross compatible and clean/easy to read.
